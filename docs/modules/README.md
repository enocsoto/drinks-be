# Módulos del sistema Drinks Billar (Backend)

Documentación por módulo: descripción, objetivos y reglas de negocio, alineada con el estilo del proyecto (ver `articulo.md` como referencia).

| Módulo    | Archivo      | Descripción breve |
|-----------|--------------|-------------------|
| Bebidas   | [BEBIDAS.md](./BEBIDAS.md)   | Catálogo de bebidas (nombre, envase, precio COP). Seed con cervezas Colombia. |
| Ventas    | [VENTAS.md](./VENTAS.md)     | Registro y consulta de ventas; detalle por bebida y vendedor. |
| Auth      | [AUTH.md](./AUTH.md)         | Login JWT y protección de rutas por rol. |
| Usuarios  | [USUARIOS.md](./USUARIOS.md) | Gestión de usuarios (ADMIN, SELLER). |
| Analítica | [ANALITICA.md](./ANALITICA.md) | Dashboard: ventas del día, por período, top vendedores. |

Cada documento incluye:
- **Descripción**: qué hace el módulo.
- **Objetivos**: para qué sirve en el negocio.
- **Reglas de negocio**: restricciones y comportamientos obligatorios.
- **Atributos / Endpoints**: referencia rápida cuando aplica.
